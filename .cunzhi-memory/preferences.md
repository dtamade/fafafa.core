# 用户偏好设置

- 用户偏好：允许并希望在合适阶段生成总结性Markdown文档、生成测试脚本、执行编译与运行测试；但仍需在关键实现方案确认后按阶段执行。
- 经寸止批示：默认同意我生成总结性Markdown报告，补充/生成测试脚本，并在需要时编译与运行（以寸止批示为准）。优先级：RFC 5952 严格化、NonBlocking 接口预留与最小Poller(select)、选项一致性与AcceptClientTimeout边界测试。
- 用户允许并偏好：生成总结性Markdown文档、生成测试脚本、执行本地编译与运行进行验证。
- 用户允许并要求：生成总结性Markdown文档、生成测试脚本、进行编译与运行验证；默认选择方向 A+B+C+D+E 做本轮推进。
- 用户偏好：每次任务结束前输出总结性Markdown文档；需要我生成测试脚本；在安全范围内帮忙编译和运行以获取验证日志。
- 用户偏好：在后续任务中，默认需要我（1）生成总结性Markdown文档，（2）生成测试脚本，（3）执行编译，（4）执行运行；并在多方案/即将结束前使用“寸止”询问确认。
- 调试日志输出请带上时间戳，便于区分和排查（例如在模块/测试的 WriteLn 调试行前加 [HH:mm:ss.SSS] 或等效前缀）。
- 文档与基准（benchmarks）要在全部功能落地完成之后再编写与运行，不要提前进行。
