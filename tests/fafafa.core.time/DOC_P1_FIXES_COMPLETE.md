### 4. ISSUE-36: Parse 解析模式行为文档 ✅

**文件**: `fafafa.core.time.parse.pas` (第 58-140 行)

**添加内容**:
详细文档化三种解析模式的行为：

#### pmStrict (严格模式)
- 要求完全匹配格式
- 分隔符、位数必须精确
- 适用于数据验证和 API 输入

#### pmLenient (宽松模式)

- 无需指定格式，自动检测
- 尝试多种常见格式
- 适用于数据清洗和多来源聚合

**性能对比**: pmStrict > pmLenient > pmSmart  
**准确性对比**: pmStrict > pmLenient > pmSmart

**影响**: 用户明确知道如何选择合适的解析模式

### 5. ISSUE-42: ISO8601 月份/年份转换文档 ✅

**文件**: `fafafa.core.time.iso8601.pas` (第 97-173 行)

**添加内容**:
详细说明 ISO 8601 Duration 的月份/年份转换限制：

**核心问题**:
- 1 年 ≠ 365 天（闰年 366 天）
- 1 月 ≠ 30 天（28-31 天不等）
- 无法精确转换为固定时长

**当前策略**:
1. **ToTDuration**: 使用近似值（1年≈365天，1月≈30天）
   - ⚠️ 误差可能达数天
2. **FromTDuration**: 只生成天/小时/分钟/秒，不生成年/月

**建议用法**:
- ✅ 精确时长：仅使用 Days/Hours/Minutes/Seconds
- ✅ 需要年/月：保持 ISO8601Duration 格式
- ⚠️ 转换带年/月的时长：明确告知用户是近似值

**示例代码**:
```pascal
// ✅ 精确转换
dur := TISO8601Duration.FromString('PT24H');
tdur := dur.ToTDuration;  // 精确

// ⚠️ 近似转换（有误差）
dur := TISO8601Duration.FromString('P1Y2M');
tdur := dur.ToTDuration;  // 近似 425 天，可能有 1-2 天误差
```

**影响**: 用户明确了解转换限制，避免误用
