# TOML Reader v2 (Pure Pascal) — Design Brief

目标
- 以纯 Pascal 实现 TOML v1.0 Reader，兼容现有 ITomlDocument 接口
- 严格语义 + 清晰错误定位 + 高性能（线性扫描、零/低拷贝）

范围
- 键与 dotted keys、表/数组表、内联表、数组
- 字符串族（基本/字面量，单行/多行、转义、续行）
- 数字（整数/浮点/指数、下划线合法性）
- 布尔、日期时间四类
- 注释与换行（CRLF/LF/CR），一致的错误定位

总体架构
- 词法器（lexer）：
  - 单次线性扫描，输出 Token 流；统一 EOL 内部表示为 `\n`（保留行/列）
  - 跳过空白与 `#` 注释；识别标识符、数值、字符串起始、符号（= , [ ] { } .）
- 语法器（parser）：
  - 递归下降/表驱动，直接构建 ITomlMutableTable/Array（无中间 AST）
  - 冲突与类型检查在写入阶段完成
- 错误模型：
  - 统一 `TTomlError`，错误码+消息+位置（Position/Line/Column）
  - 常见错误：重复键、类型冲突、未闭合字符串/内联表/数组、非法数字格式

性能要点
- 指针/索引推进、避免中间字符串拷贝
- 对字符串/数字做“就地切片+按需解码”

里程碑（建议）
1) 词法器基础（Token、注释/空白、符号、标识符）+ 单测
2) 最小语法（键值赋值、dotted keys、[table]、[[table]]) + 冲突检测
3) 字符串族（单/多行、转义、续行）+ 负例
4) 数字严格度（整数/浮点/指数、下划线）+ 负例
5) 日期时间四类 + Roundtrip
6) 内联表/数组嵌套 + 负例
7) 回归与性能（bench 支持 parser=native-v2 切换）

与现版本兼容
- 保持 ITomlDocument/ITomlTable/ITomlArray 接口不变
- 错误定位策略文档化，便于一致性测试

开放问题
- 多行基本字符串续行与未知转义的容错策略（严格报错 vs. 宽松保留）
- 大文本的增量解析（后续优化）
