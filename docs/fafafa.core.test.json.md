# fafafa.core.test JSON Report (Draft)

This document describes the JSON report generated by the test framework's JSON listener.

## How to enable

- Command line (runner):
  - --json=path           Write JSON report to the given path
  - Optional isolation:   --no-console --no-junit
- Example (from repo root on Windows PowerShell):
  - .\tests\fafafa.core.fs\bin\tests.exe --filter=fs --json=.\tests\fafafa.core.fs\bin\report.json --no-console --no-junit

## Schema (minimal)

Root object:
- name: string                 // test suite logical name (e.g. "fafafa.core.test")
- timestamp: string            // ISO-like time (yyyy-mm-ddThh:nn:ssZ)
- hostname: string             // machine host name
- tests: TestCase[]            // array of test cases (including subtests)
- total: number                // how many test events recorded (incl. subtests)
- failures: number             // how many failed
- elapsed: number              // total time in seconds

TestCase object:
- classname: string            // parent path of the test (everything before last '/'; 'default' if empty)
- name: string                 // last segment of the test name
- time: number                 // elapsed seconds
- status: 'passed' | 'failed'
- message?: string             // present only on failure

## Notes

- The listener is backend-agnostic via IJsonReportWriter. Currently it uses RTL (fpjson); once the project's internal JSON module is ready, you can provide a new implementation and switch the factory in the runner.
- The report is designed to be simple and friendly to downstream tools (CI parsers, dashboards).
- All strings are written as-is; escaping is handled by the JSON writer.

## Future extensions (optional)

- skipped support (status: 'skipped', with reason)
- file/line metadata when available
- properties/counters per test
