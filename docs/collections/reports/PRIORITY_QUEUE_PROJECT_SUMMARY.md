generic TPriorityQueue<T> = record

  function Dequeue: T;                    // O(log n)
  function Peek: T;                       // O(1)

end;

## ⚠️ 风险和注意事项

### 已识别风险

1. **任务重新调度问题** ⚠️
   - **风险**: 当任务的 NextRunTime 改变时，需要从队列中移除并重新插入
   - **缓解**: 在 ProcessTasks 中正确处理任务时间更新

2. **接口变化影响** ⚠️
   - **风险**: TList 到 TPriorityQueue 的API差异可能导致编译错误
   - **缓解**: 按集成指南逐步修改，确保所有调用点都更新

3. **性能测试覆盖** ⚠️
   - **风险**: 可能存在未预见的性能瓶颈
   - **缓解**: 进行全面的性能基准测试，包括边界情况

4. **线程安全** ⚠️
   - **风险**: 优先队列本身不是线程安全的
   - **缓解**: 确保所有操作都在锁保护下进行 (已有 FLock)

### 关键注意事项

- ✅ 优先队列需要显式初始化 (`Initialize`)
- ✅ 比较器函数必须全局或静态
- ✅ 任务时间更新需要重新入队
- ✅ Remove 操作仍是 O(n)，但实际场景中使用不频繁
