# 🚀 fafafa.core.benchmark v2.1+ 高质量迭代路线图

## 📋 项目状态评估

### ✅ **v2.0 已完成成就**
- Google Benchmark 风格的 State-based API
- 完整的基准测试框架架构
- 专业的输出格式和表格对齐
- 测试夹具支持
- 自定义计数器和复杂度分析
- 50+ 个测试示例验证功能

### ⚠️ **发现的关键问题**
经过深入分析，发现当前实现存在**严重的测量准确性问题**：

1. **计时逻辑错误** - 在构造函数中开始计时，包含初始化开销
2. **关键功能缺失** - PauseTiming/ResumeTiming 是空实现
3. **缺乏科学测量流程** - 没有预热、校准、统计分析
4. **固定迭代次数** - 不根据实际执行时间调整

**结论：当前的测量结果可能不可靠，需要立即修复！**

## 🎯 v2.1 版本目标：测量准确性修复

### 🔥 **P0 级别 - 必须立即修复（v2.1.0）**

#### **1. 修复计时逻辑错误**
- **问题**：`FStartTime := FTick.GetCurrentTick;` 在构造函数中执行
- **修复**：在第一次 `KeepRunning()` 调用时开始计时
- **影响**：消除初始化开销对测量结果的影响

#### **2. 实现 PauseTiming/ResumeTiming**
- **问题**：当前是空实现，用户调用无效果
- **修复**：实现真正的计时暂停和恢复机制
- **实现**：
  ```pascal
  private
    FTimingPaused: Boolean;
    FPausedTime: UInt64;
    FTotalPausedDuration: UInt64;
  ```

#### **3. 添加预热阶段**
- **问题**：直接开始测量，冷启动效应影响结果
- **修复**：在正式测量前运行预热迭代
- **实现**：默认5次预热迭代，可配置

#### **4. 修复 KeepRunning 计时开始点**
- **问题**：计时开始时机不正确
- **修复**：确保在用户代码开始执行时才开始计时

### 🚀 **P1 级别 - 重要功能（v2.1.1-v2.1.3）**

#### **5. 实现校准算法**
- **目标**：动态确定合适的迭代次数
- **算法**：
  1. 从1次迭代开始
  2. 测量执行时间
  3. 如果时间太短，迭代次数翻倍
  4. 重复直到达到目标测量时间（默认1ms）

#### **6. 实现目标时间控制**
- **功能**：根据目标测量时间调整迭代次数
- **配置**：允许用户设置目标测量时间
- **默认**：1000ms 总测量时间

#### **7. 添加多次运行统计**
- **功能**：运行多次基准测试，计算统计数据
- **统计**：平均值、标准差、最小值、最大值
- **默认**：运行5次取平均值

#### **8. 实现智能迭代次数估算**
- **算法**：指数增长 + 二分查找优化
- **目标**：找到最优的迭代次数
- **约束**：单次测量时间 >= 1ms，总时间 <= 目标时间

#### **9. 添加测量稳定性检查**
- **功能**：检测测量结果的一致性
- **指标**：变异系数（CV）< 5%
- **处理**：如果不稳定，增加迭代次数或报告警告

## 🎯 v2.2 版本目标：专业性提升

### 🎨 **P2 级别 - 进一步优化（v2.2.0-v2.2.3）**

#### **10. 实现异常值检测和过滤**
- **算法**：IQR方法或Z-score方法
- **处理**：自动排除异常测量结果
- **报告**：显示异常值数量和处理情况

#### **11. 添加置信区间计算**
- **统计**：95%置信区间
- **显示**：在输出中显示置信区间
- **用途**：提供统计学上的可信度

#### **12. 实现参数化基准测试便利API**
- **目标**：简化多参数测试的编写
- **API设计**：
  ```pascal
  RegisterParameterizedBenchmark('BM_Sort', @BM_Sort_Parameterized)
    .WithArgs([10, 50, 100, 500, 1000]);
  ```

#### **13. 添加命令行参数处理**
- **功能**：支持过滤、格式选择、重复次数等
- **参数**：
  - `--filter=pattern` - 过滤基准测试
  - `--repetitions=N` - 重复次数
  - `--warmup=N` - 预热次数
  - `--min-time=N` - 最小测量时间

#### **14. 实现线程安全的多线程基准测试**
- **功能**：支持并发性能测试
- **API**：`RegisterBenchmark(...).Threads(4)`
- **测量**：每线程性能 + 总体吞吐量

#### **15. 添加内存使用统计**
- **功能**：跟踪内存分配和释放
- **指标**：峰值内存、平均内存、分配次数
- **集成**：与现有计数器系统集成
