# fafafa.core.fs 性能优化计划

## 🎯 优化目标

基于基准测试结果，制定系统性的性能优化策略，目标是将关键操作性能提升2-10倍。

## 📊 当前性能基线（需要重新测量）

使用 GetTickCount64 重新测量后的准确基线：

| 操作类型 | 当前性能 | 目标性能 | 优化倍数 |
|---------|---------|---------|---------|
| 目录扫描 | TBD ms/op | <1ms/op | 10x |
| 文件创建 | TBD ms/op | <0.1ms/op | 5x |
| 文件读取 | TBD MB/s | >500MB/s | 3x |
| 路径标准化 | TBD ops/s | >1M ops/s | 2x |

## 🚀 优化策略

### 1. 目录扫描优化 (最高优先级)

**问题分析**:
- 当前使用 FindFirst/FindNext，每次调用都有系统调用开销
- 字符串转换开销大
- 内存分配频繁

**优化方案**:
- **Windows**: 直接使用 FindFirstFileW/FindNextFileW API
- **Unix**: 使用 opendir/readdir 替代高级封装
- **缓存优化**: 预分配 TStringList 容量
- **批量处理**: 一次性读取多个条目

### 2. 文件I/O优化

**问题分析**:
- 高级封装层次过多
- 缓冲区大小不优化
- 频繁的内存分配/释放

**优化方案**:
- **直接API调用**: 绕过高级封装，直接使用系统API
- **缓冲区优化**: 使用更大的缓冲区（64KB-1MB）
- **内存池**: 实现缓冲区内存池，减少分配开销
- **异步I/O**: 对大文件使用异步读写

### 3. 路径操作优化

**问题分析**:
- 字符串操作频繁
- 动态数组重新分配
- 不必要的标准化步骤

**优化方案**:
- **字符串缓存**: 预分配字符串缓冲区
- **原地操作**: 减少字符串复制
- **快速路径**: 对简单路径使用快速算法
- **缓存结果**: 缓存常用路径的标准化结果

### 4. 内存管理优化

**优化方案**:
- **对象池**: 实现 TFsFile 对象池
- **缓冲区池**: 实现可重用的缓冲区池
- **栈分配**: 对小缓冲区使用栈分配
- **延迟初始化**: 按需初始化资源

### 5. 系统调用优化

**优化方案**:
- **批量操作**: 合并多个系统调用
- **缓存元数据**: 缓存文件状态信息
- **减少转换**: 减少字符串编码转换
- **直接调用**: 绕过RTL，直接调用系统API

## 📈 性能测试计划

### 测试场景
1. **微基准测试**: 单个操作的精确测量
2. **宏基准测试**: 真实应用场景模拟
3. **压力测试**: 大量文件/大文件处理
4. **并发测试**: 多线程性能测试

### 测试指标
- **延迟**: 单次操作时间
- **吞吐量**: 每秒操作数/数据传输率
- **内存使用**: 峰值内存和内存分配次数
- **CPU使用**: CPU利用率和缓存命中率
