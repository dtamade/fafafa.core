# fafafa.core.mem 项目总结
> 说明：本文档为阶段性记录，内容可能与当前代码不一致；以 `docs/fafafa.core.mem.md` 与 `tests/fafafa.core.mem/README.md` 为准。

## 🎯 项目概述

`fafafa.core.mem` 是 fafafa 框架的核心内存管理模块，采用门面模式设计，提供统一的内存操作接口和多种高性能内存池实现。

## 🏗️ 架构成就

### 设计理念
- **中规中矩** - 遵循传统设计模式，避免过度复杂
- **门面模式** - 简洁的统一入口，隐藏实现细节
- **职责分离** - 每个模块功能单一，易于维护
- **工业级质量** - 参考 nginx 等成熟项目的设计

### 模块架构
```
fafafa.core.mem/
├── fafafa.core.mem.pas           # 主门面模块 (140行)
├── fafafa.core.mem.memPool.pas   # 通用内存池 (200行)
├── fafafa.core.mem.stackPool.pas # 栈式内存池 (180行)
└── fafafa.core.mem.pool.slab.pas  # nginx风格Slab (393行)
```

## 🚀 核心特性

### 1. 统一的内存操作接口
- 重新导出 `fafafa.core.mem.utils` 的内存操作函数
- 重新导出 `fafafa.core.mem.allocator` 的分配器类型
- 提供一致的 API 体验

### 2. 多种内存池实现

#### TMemPool - 通用内存池
- **用途**: 固定大小对象的高效分配
- **特点**: 预分配、O(1)操作、容量控制
- **场景**: 频繁分配/释放固定大小对象

#### TStackPool - 栈式内存池
- **用途**: 临时对象的顺序分配
- **特点**: 栈式管理、状态保存/恢复、批量释放
- **场景**: 解析器、编译器、临时计算

#### TSlabPool - nginx风格Slab分配器
- **用途**: 多种大小对象的高效分配
- **特点**: 页面管理、大小类别、位图跟踪
- **场景**: 服务器程序、网络应用、系统软件

### 3. 性能优化
- **时间复杂度**: 所有操作都是 O(1)
- **内存效率**: 减少碎片，提高利用率
- **对齐支持**: 满足硬件对齐要求

## 📊 技术亮点

### nginx风格Slab分配器
参考 nginx 的成熟设计，实现了：
- **页面管理**: 4KB页面作为基本单位
- **大小类别**: 9个预定义类别 (8-2048字节)
- **位图管理**: 高效的空闲块跟踪
- **链表管理**: 空闲页面和大小类别链表

### 统一的分配器类型
所有内存池都使用 `TAllocator` 类型，保持架构一致性：
```pascal
constructor Create(...; aAllocator: TAllocator = nil);
```
