## 🔧 技术要点

### 事件类型选择
- **自动重置事件**：适合一对一通信，信号被消费后自动重置
- **手动重置事件**：适合一对多通信，需要手动重置信号

### 常见使用模式
1. **信号通知**：一个线程通知另一个线程某个条件已满足
2. **资源保护**：控制对共享资源的访问
3. **任务协调**：协调多个线程的执行顺序
4. **状态同步**：同步不同线程间的状态变化

### 性能考虑
- 避免不必要的等待操作
- 合理选择超时时间
- 减少锁竞争
- 优化临界区大小

## 🐛 常见问题

### Q: 什么时候使用自动重置 vs 手动重置？
**A:** 
- **自动重置**：一个信号只唤醒一个等待线程，适合任务分发
- **手动重置**：一个信号可以唤醒所有等待线程，适合状态广播

### Q: 如何避免死锁？
**A:**
- 始终以相同顺序获取多个锁
- 使用超时等待避免无限阻塞
- 及时释放不再需要的锁
- 避免在持有锁时调用可能阻塞的操作

### Q: 如何优化性能？
**A:**
- 减少不必要的同步操作
- 使用 TryWait 避免阻塞
- 合理设置等待超时
- 考虑使用无锁数据结构
