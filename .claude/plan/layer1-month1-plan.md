## 📅 第 1 个月详细计划（Week 1-4）

### **Week 1：Layer 1 测试补充（优先级：最高）**

**目标**：将测试覆盖率从 70% 提升到 85%+

#### 任务清单

**1. 现代 API 测试补充**（3 天）
- [ ] 补充 OnceLock 测试（初始化、多线程竞争、错误处理）
- [ ] 补充 LazyLock 测试（懒加载、多线程安全、性能验证）
- [ ] 补充 Parker 测试（park/unpark、超时、多线程场景）
- **验收标准**：每个模块至少 15 个测试用例，覆盖基本功能、并发场景、错误处理

**2. 超时测试补充**（2 天）
- [ ] 补充 TryLockFor 测试（正常超时、边界条件、取消场景）
- [ ] 补充 TryLockUntil 测试（绝对时间超时、时钟变化处理）
- [ ] 补充 WaitFor/WaitUntil 测试（条件变量超时场景）
- **验收标准**：超时测试覆盖率达到 80%+，包含边界条件测试

**3. 错误路径测试补充**（2 天）
- [ ] 补充 Poison 机制测试（Mutex/RWLock 的 Poison 状态传播）
- [ ] 补充异常处理测试（资源耗尽、无效参数、死锁检测）
- [ ] 补充跨进程同步测试（命名原语的错误场景）
- **验收标准**：错误路径测试覆盖率达到 60%+

**交付物**：
- 新增测试用例 50+ 个
- 测试覆盖率报告（目标：85%+）
- 测试失败问题清单（如有）
